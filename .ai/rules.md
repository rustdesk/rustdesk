# Regras do Projeto

- usar exclusivamente servidor privado
- impedir fallback para servidores públicos
- não ler configurações do nome do executável
- ignorar configurações .toml locais
- access_mode deve ser sempre confirmação obrigatória
- desabilitar unattended access
- desabilitar one-time password
- desabilitar permanent password
- direct_server deve permanecer false
- UI não deve permitir alterar networking
- manter compatibilidade Flutter + Rust
- não alterar protocolo RustDesk
